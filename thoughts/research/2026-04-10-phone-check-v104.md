# Ресёрч: Проверка DNS с телефона пользователя v1.0.4

**Дата:** 2026-04-10

## Задача
Пользователь хочет нажать кнопку на сайте (с iPhone в той же WiFi что Xbox) и узнать — будет ли DNS работать — ДО того как прописывать его в Xbox.

## Ограничения браузера (критично)

Браузерный JavaScript **не может**:
1. Делать произвольные DNS-запросы к серверу по UDP/53 (стандартный DNS) — заблокировано из соображений безопасности
2. Изменять DNS-настройки устройства
3. Делать кросс-доменные запросы без CORS к чужим серверам
4. Напрямую проверять "работает ли этот DNS для Xbox"

## Что технически возможно

### Вариант 1: DoH (DNS over HTTPS) через `fetch`
- Только если DNS-сервер поддерживает DoH И имеет CORS-заголовки
- Cloudflare (`1.1.1.1`), Google (`8.8.8.8`) — поддерживают DoH, но CORS разрешён только для своих страниц
- **xbox-dns.ru** заявляет DoH endpoint `https://xbox-dns.ru/dns-query` — НО CORS у них нет (проверено запросом)
- **Большинство Xbox-bypass DNS (176.99.11.77, 45.90.33.120 и др.) не поддерживают DoH вообще** — это классические DNS-over-UDP серверы
- **Вывод:** Проверить произвольный DNS из браузера нельзя

### Вариант 2: Проверка xsts IP через `fetch` в режиме `no-cors`
- xsts IP — это обычные HTTPS-прокси на порту 443
- `fetch('https://50.7.85.221/', {mode: 'no-cors'})` — браузер не блокирует сам запрос, но и не даёт читать ответ
- **Важно:** в режиме `no-cors` мы НЕ узнаем статус (200/404/и т.д.), но узнаем **достучались ли вообще**
- Если fetch резолвится без ошибки → IP доступен с телефона пользователя
- Если таймаут/network error → IP заблокирован провайдером / недоступен
- **Это работает!** Можем проверить все 15 xsts IP прямо с iPhone пользователя
- **Ценность высокая:** пользователь в РФ увидит "из твоей сети 50.7.85.221 доступен, 164.90.228.248 заблокирован"

### Вариант 3: Image tag trick
- `<img src="https://50.7.85.221/favicon.ico">` → onload/onerror
- Аналогично варианту 2, но через `<img>` — работает даже в старых браузерах
- Преимущество: не нужен CORS вообще
- **Это тоже работает для xsts IP**

### Вариант 4: Проверка xsts домена через публичный DoH
- `fetch('https://cloudflare-dns.com/dns-query?name=xsts.auth.xboxlive.com&type=A', {headers: {'accept': 'application/dns-json'}})`
- Cloudflare DoH имеет CORS → работает из любого сайта
- Вернёт что xsts резолвится у Cloudflare (обычно Azure IP)
- **НО:** это резолв через Cloudflare, а не через наш bypass-DNS. Не говорит нам работает ли bypass.
- **Полезно для другого:** показать пользователю "вот что отвечает обычный DNS на xsts" — для сравнения

## Почему DNS прописываются парами?

**Не из-за какой-то парной логики** — это просто redundancy. Secondary — бэкап если primary не отвечает. Xbox использует primary, при сбое переключается на secondary. **Проверять можно только primary — этого достаточно.**

## Главный вывод

**Проверка произвольного DNS с телефона невозможна.** Браузер не позволяет делать DNS-запросы к серверам.

**НО есть три работающих кейса:**

1. **Проверка xsts IP с телефона** (вариант 2/3) — **ВЫСОКАЯ ЦЕННОСТЬ**
   - Из сети пользователя проверяем достижим ли прокси
   - Решает половину проблемы: xsts IP проверку из GitHub Actions (v1.0.3) мы уже сделали, но из сети пользователя результат может отличаться (провайдер может блокировать)
   - 5-10 строк JS, встраивается в кнопку "Проверить с моего телефона"

2. **Проверка xsts через Cloudflare DoH** — **НИЗКАЯ ЦЕННОСТЬ**
   - Показывает что возвращает обычный DNS (Cloudflare) для xsts
   - Полезно только как "посмотреть нормальный ответ для сравнения"

3. **Полная проверка DNS с телефона** — **НЕВОЗМОЖНО**
   - Требует либо поддержки DoH+CORS у всех наших DNS (их нет), либо нативного API браузера (нет такого API)
   - **Альтернатива:** социальный трекер — мы уже его сделали (кружки пользователей). Friends отмечают у кого что работает, другие видят.

## Рекомендация

**Реализовать только "Проверка xsts IP с телефона"** (вариант 2).

- Кнопка "Проверить с моего устройства" на xsts-карточках
- Делаем `fetch(https://IP/, {mode:'no-cors'})` с таймаутом 5 сек
- Показываем `✓ Доступен с твоего устройства` или `✗ Недоступен из твоей сети`
- Это дополняет серверную проверку (v1.0.3) — теперь видно статус и со стороны сервера, и со стороны пользователя

**Для DNS-серверов (IPv4/IPv6) — не реализовывать вообще.** Вместо этого улучшить социальный трекер (кружки пользователей) — когда друг отмечает "работает", это самая надёжная валидация.
