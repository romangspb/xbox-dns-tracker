## 3. Технический подход

### Извлечение данных

**Telegram (t.me/s/):**
- GET запрос на `https://t.me/s/{channel_name}`
- BeautifulSoup → `.tgme_widget_message_text` для текста
- `<time datetime="...">` для даты поста
- Regex `r'\b(\d{1,3}\.){3}\d{1,3}\b'` для IP-адресов
- Контекст вокруг IP для определения типа (primary/secondary/xsts)

**GitHub README (chipslays):**
- GET `https://raw.githubusercontent.com/chipslays/0x80a40401/main/README.md`
- Парсинг markdown-таблицы regex: `\|.*\|.*\|` паттерн
- Маппинг столбцов: Основной, Дополнительный, Статус

**Сайты (xbox-dns.ru, xbox-news.ru):**
- httpx GET с реалистичным User-Agent
- BeautifulSoup + CSS-селекторы
- xbox-dns.ru: `code` теги в `#setup`
- xbox-news.ru: `.entry-content` → regex по IP

### Нормализация и дедупликация

```python
import ipaddress
normalized = str(ipaddress.ip_address(raw_ip.strip()))
```

Дедупликация по нормализованной паре (primary, secondary). Если DNS появился из нескольких источников — объединяем source_urls.

### Классификация

| type | Определение |
|------|------------|
| dns_pair | Пара IP (primary + secondary) для настроек Xbox |
| xsts_ip | IP для xsts.auth.xboxlive.com (подмена на роутере) |
| vpn | Любой VPN-способ |
| router | Настройка на роутере (static host, DNS) |
| other | Всё остальное |

**difficulty:** easy / medium / hard — по типу (dns_pair = easy, router = medium, vpn/vps = hard).

### Фильтр по давности

- Telegram: дата из `<time>` тега. Если > 2 месяцев — не включать как новый.
- GitHub: нет дат у отдельных записей → весь файл как один источник, дата = last commit.
- Сайты: дата из HTML meta или текста статьи.

## 4. Кто уже решал эту задачу?

**Специализированных парсеров Xbox DNS нет.** Найдено:

1. **Steelio/Telegram-Post-Scraper** — generic scraper для t.me/s/ через requests + BeautifulSoup. Подход совпадает с нашим, но нет извлечения IP.

3. **skorches/xbox-dns-bypass** — bash-скрипты для валидации, не для парсинга.
4. **xbox-dns.ru** — сервис предоставляет свои DNS, но не агрегирует чужие.

**Вывод:** Прямых аналогов нет. Наш проект — первый автоматический агрегатор.

## 5. Выбор библиотек

| Задача | Библиотека | Почему |
|--------|-----------|--------|
| HTTP | httpx | Async-ready, timeout из коробки, замена requests |
| HTML-парсинг | BeautifulSoup4 | Стандарт, отказоустойчивый парсинг |
| IP-нормализация | ipaddress (stdlib) | Стандартная библиотека, валидация + нормализация |
| DNS (v0.2) | dnspython | Для будущей автопроверки |
| JSON | json (stdlib) | Стандартная библиотека |
| Дата/время | datetime (stdlib) | Стандартная библиотека |

**Альтернативы рассмотренные и отклонённые:**
- **Telethon (Telegram API):** Мощнее, но требует API credentials + аккаунт. Overkill для 1 запроса/сутки на публичные каналы.
- **Scrapy:** Framework для больших проектов. У нас 5 простых запросов — httpx достаточно.
- **Selenium/Playwright:** Для JS-рендера. Наши источники отдают HTML без JS — не нужно.

## 6. Рекомендация

**Лучший подход:** Модульный парсер с отдельным классом/функцией на каждый тип источника (telegram, github_readme, website). Общий пайплайн:

```
fetch → extract_raw → normalize_ips → classify → deduplicate → output_json
```

**Почему этот подход лучший:**
1. Источники разные по формату — общий парсер невозможен.
2. Модульность позволяет добавлять/убирать источники без правки ядра.
3. JSON-файл в репо — читаемые диффы, нет внешних зависимостей (БД).
4. Дедупликация через ipaddress — надёжна и проста.

## 7. Риски и edge cases

1. **t.me/s/ отдаёт 429 или пустую страницу** → retry с backoff, статус "не спарсен" вместо падения.
2. **Канал стал приватным** → 302 redirect → обработать, пометить источник неактивным.
3. **HTML структура t.me изменилась** → CSS-селекторы в конфиг, алерт при 0 результатов.
4. **IP в нестандартном формате** (ведущие нули, пробелы) → ipaddress нормализует.
5. **Вредоносные DNS** → пометка "непроверенный" если 1 источник (проверка в v0.2).
6. **0.0.0.0 как secondary** → валидный паттерн (означает "нет secondary"), хранить as-is.
7. **IPv6 адреса** → поддержать, ipaddress работает с ними.
