# Ресёрч: Веб-страница + GitHub Actions v1.0

**Дата:** 2026-04-10

## GitHub Actions

**Cron:** `schedule: cron: '37 4 * * *'` — раз в день в нечётное время (UTC). Задержки 5-20 мин — норма.

**Ручной запуск:** `workflow_dispatch` — кнопка "Run workflow" в GitHub UI. Для 5 друзей достаточно ссылки на страницу Actions.

**Workflow:** checkout → setup-python → pip install → python -m src → commit data.json → deploy Pages.

**Deploy:** `actions/upload-pages-artifact` + `actions/deploy-pages` — встроенный способ GitHub. Альтернатива: `peaceiris/actions-gh-pages` в ветку gh-pages.

## GitHub Pages для private repo

GitHub Pages доступен для private repos на бесплатном плане (с 2024). Настройка: Settings → Pages → Source: GitHub Actions.

**Ограничение:** сайт всё равно будет публично доступен по URL (github.io). Для 5 друзей — ок.

## Фронтенд

**Стек:** vanilla HTML + CSS + JS. Без фреймворков. `fetch('data.json')` для данных.

**Из ux.md:**
- iPhone 16: 393×852, viewport-fit=cover
- Тёмная тема: #121212 bg, палитра в CSS variables
- Копирование DNS одним тапом (ClipboardItem API для Safari)
- 5 кружков-статусов под каждым DNS (зелёный/красный/серый)
- Группировка по сложности: вертикальные секции с цветной полоской слева
- Модальное окно выбора пользователя при первом визите, localStorage

## Кнопка "Обновить сейчас" / "Перепроверить DNS"

Для 5 друзей: ссылка на GitHub Actions → "Run workflow". Просто и надёжно. API-интеграция — overkill.

## Структура деплоя

```
site/
├── index.html
├── style.css
├── app.js
└── data.json  (копируется из data/ при деплое)
```

GitHub Actions: парсер генерирует data/data.json → копируется в site/data.json → деплоится на Pages.
