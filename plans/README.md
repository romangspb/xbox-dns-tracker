# Технические планы реализации

Папка `plans/` — место для технических планов реализации функций проекта.

## Правила

1. **Один план = одна функция.** Если план уже есть — работаем с ним.
2. **Формат имени файла:** `YYYY-MM-DD-название-функции.md`
3. Каждый план делится на **фазы**. У каждой фазы статус `[ ]` или `[x]`.
4. В конце плана — итоговый блок: реализован целиком или нет, что осталось.
5. Любой агент, работающий с планом, обязан его актуализировать: отмечать сделанные фазы, дописывать что сделал.

## Формат фазы

```
## Фаза N: Название [ ]

### Задачи
- [ ] Техническая задача 1
- [ ] Техническая задача 2

### Что изменится (для приёмки)
Было: описание текущего состояния.
Стало: описание результата на языке пользователя.

### Итог реализации (заполняется ПОСЛЕ завершения фазы)
Что сделано: какие файлы созданы/изменены, какие решения приняты и почему.
Что отличается от плана: если пришлось отойти — что и почему.
Известные проблемы: что не доделано, что нужно вернуться и доработать.
```

Блок «Итог реализации» обязателен при закрытии фазы — это технический лог для следующей сессии.
