# Спецификация: Парсер v0.1

**Дата:** 2026-04-10
**Версия:** v0.1
**Статус:** draft

---

## Проблема

5 пользователей Xbox в РФ вручную ищут рабочие DNS для обхода ошибки 0x80a40401 по разрозненным Telegram-каналам, GitHub-репозиториям и сайтам. Это занимает время, информация дублируется, устаревшие DNS не отсеиваются.

## Цель

Автоматически собрать DNS-пары и способы обхода из 6 источников в единый нормализованный JSON-файл, готовый для отображения на веб-странице (v1.0).

## Scope (что входит)

1. Парсинг 6 источников:
   - t.me/s/xboxnews_ru (Telegram)
   - t.me/s/xbox_dns (Telegram, замена xbox0games)
   - github.com/chipslays/0x80a40401 (GitHub README)
   - xbox-dns.ru (сайт)
   - xbox-news.ru/news/1049425/ (статья)
   - github.com/skorches/xbox-dns-bypass (GitHub)

2. Извлечение данных: DNS-пары (primary + secondary), xsts IP, способы обхода

3. Нормализация IP через `ipaddress` (убирает ведущие нули, пробелы, валидирует)

4. Дедупликация по нормализованной паре (primary, secondary)

5. Классификация:
   - **По типу:** dns_pair, xsts_ip, vpn, router, other
   - **По сложности:** easy, medium, hard

6. Фильтр по давности: записи из постов старше 2 месяцев не включаются как новые

7. Жизненный цикл записи:
   - Новая запись → active: true
   - Не встречается 2 месяца → active: false (не удаляется)
   - Появилась снова → recheck: true (флаг "перепроверить")

8. Выходной файл: `data/data.json` — единая точка данных для всего проекта

9. Локальный запуск: `python -m src.main` из корня проекта

## Non-Goals (что НЕ входит)

- Автопроверка DNS (v0.2)
- Веб-страница (v1.0)
- Telegram-бот (v1.1)
- GitHub Actions (v1.0)
- Персональный трекинг статусов
- Парсинг комментариев и обсуждений
- Поиск новых источников на лету

## Acceptance Criteria

1. **AC-1:** При запуске `python -m src.main` парсер обходит все 6 источников и создаёт/обновляет `data/data.json`
2. **AC-2:** `data.json` соответствует схеме из technical.md (methods[], sources[], updated_at)
3. **AC-3:** Все IP нормализованы через `ipaddress` — нет дубликатов с разным форматом
4. **AC-4:** Каждый метод классифицирован по type (dns_pair/xsts_ip/vpn/router/other) и difficulty (easy/medium/hard)
5. **AC-5:** Записи из постов старше 2 месяцев не включаются
6. **AC-6:** Если источник недоступен (429, таймаут, 302) — парсер не падает, пропускает и логирует
7. **AC-7:** При повторном запуске: существующие данные обновляются (last_seen), новые добавляются, дубли не создаются
8. **AC-8:** IPv6 адреса обрабатываются корректно наравне с IPv4
9. **AC-9:** 0.0.0.0 как secondary DNS — валидный паттерн, сохраняется

## Constraints

- Python 3.12+
- Зависимости: httpx, beautifulsoup4, lxml (опционально для скорости BS4)
- Без внешних API-ключей (Telegram через t.me/s/, GitHub через raw.githubusercontent.com)
- JSON-файлы в репозитории, без БД
- Один запуск = один проход по всем источникам (без daemon/schedule)

## Схема data.json

```json
{
  "updated_at": "ISO 8601",
  "methods": [
    {
      "id": "dns-{hash}",
      "type": "dns_pair | xsts_ip | vpn | router | other",
      "difficulty": "easy | medium | hard",
      "primary_dns": "IP или null",
      "secondary_dns": "IP или null",
      "description": "текст если не dns_pair",
      "sources": ["source-id-1", "source-id-2"],
      "source_urls": ["https://..."],
      "first_seen": "YYYY-MM-DD",
      "last_seen": "YYYY-MM-DD",
      "active": true,
      "recheck": false,
      "instruction_url": null
    }
  ],
  "sources": [
    {
      "id": "src-{n}",
      "name": "отображаемое имя",
      "type": "telegram | github | website",
      "url": "https://...",
      "last_parsed": "ISO 8601",
      "status": "ok | error | unavailable",
      "error_message": null
    }
  ]
}
```

## Метрики успеха

- Парсер находит минимум 10 уникальных DNS-пар из всех источников
- Все 6 источников обрабатываются без ошибок (при их доступности)
- Повторный запуск не создаёт дублей
- Время выполнения < 60 секунд
