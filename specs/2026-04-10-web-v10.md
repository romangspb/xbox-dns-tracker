# Спецификация: Веб-страница + GitHub Actions v1.0

**Дата:** 2026-04-10

## Проблема

Парсер работает только локально. Данные в data.json, но пользователи не видят их — нет интерфейса. Запуск парсера требует терминала.

## Цель

Мобильная веб-страница на GitHub Pages, где 5 пользователей видят актуальные DNS, их статус проверки, могут скопировать DNS одним тапом и отметить "работает / не работает". Парсер запускается автоматически раз в сутки через GitHub Actions.

## Scope

1. **Статичная страница** (site/index.html + style.css + app.js)
   - Тёмная тема, iPhone 16 first
   - Загрузка data.json через fetch
   - DNS сгруппированы по сложности (easy → medium → hard)
   - Для каждого DNS: IP, статус проверки (working/unsafe/timeout...), источники
   - Кнопка копирования DNS одним тапом
   - Персональный трекинг: 5 кружков (Рома, Дима, Саша, Макс, Влад)
   - Модальное окно выбора пользователя при первом визите
   - Ссылка "Обновить сейчас" → GitHub Actions (workflow_dispatch)

2. **GitHub Actions workflow** (.github/workflows/update.yml)
   - Триггеры: cron ежедневно + workflow_dispatch
   - Шаги: checkout → python → pip install → python -m src → deploy Pages

3. **Deploy на GitHub Pages** через actions/deploy-pages

## Non-Goals

- Telegram-бот (v1.1)
- Серверный бэкенд
- Авторизация / API-ключи на фронтенде
- Синхронизация статусов между устройствами (localStorage only)

## Acceptance Criteria

1. **AC-1:** Страница доступна по https://romangspb.github.io/xbox-dns-tracker/
2. **AC-2:** DNS отображаются сгруппированные по difficulty (easy/medium/hard)
3. **AC-3:** Для каждого DNS видны: IP, статус проверки, количество источников
4. **AC-4:** Тап на DNS → копирует в буфер, feedback "Скопировано"
5. **AC-5:** Тап на свой кружок → переключает статус (работает/не работает/не пробовал)
6. **AC-6:** Статусы сохраняются в localStorage между визитами
7. **AC-7:** GitHub Actions запускается по cron раз в сутки и обновляет data.json
8. **AC-8:** Страница корректно отображается на iPhone 16 (393px width)
9. **AC-9:** Тёмная тема по brand-guidelines (#121212, палитра CSS)
